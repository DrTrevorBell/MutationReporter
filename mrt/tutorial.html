Introduction

This tutorial discusses the "Mutation Reporter Tool", using example data from the hepatitis B virus (HBV). The tool can be used with aligned sequence data (nucleotides or amino acids) from any source, however. The HBV genome, which is circular, is approximately 3200 nucleotides in length and has been well-characterized. Mutations (single nucleotide polymorphisms) in the genetic sequence of HBV are common. Certain patterns of mutations are characteristic of a particular genotype or subgenotype. Patterns at other loci are characteristic of known drug-resistant mutants or indicate other important viral characteristics, such as down-regulation of a viral antigen.

Several mutations of interest are located in the basic core promoter/precore (BCP/PC) region of the viral genome. This region of the genome is located between genomic co-ordinates 1606 and 1941 (depending on genotype/subgenotype). Mutations in this region are known by their genomic co-ordinates. For example, the variable Kozak sequence is located at co-ordinates 1809-1812 of the viral genome. Other loci of interest are found at positions 1762, 1764, 1858, 1862, 1888 ad 1896. Sequence data at other loci in this region are typically well-conserved and not of interest.

The "Mutation Reporter Tool" extracts and summarizes nucleotide or amino acid sequence data at known loci of interest, making it easier to identify patterns and mutations of interest, as data from sites of no interest are not included.

The tool is an online service and requires an input file to be uploaded. Output from the tool is displayed on a web-page. Certain components of the output can be downloaded.

Parameters found on the input web-page of the tool, which can be specified by the user, are indicated below in this typeface.

Input Parameters

The tool requires a plain text file in FASTA format as input. This file should contain aligned sequence data (nucleotide or amino acid) for the samples to be processed. The sequence data type (Nucleotides or Amino Acids) should be specified on the input page.

Whilst the tool can process sequence data of virtually any length, typically sequence data from PCR amplicons (fragments) would be used. The BCP/PC region of HBV is sequenced routinely, generating a PCR amplicon of approximately 500 nucleotides. Known loci of interest in this fragment are labeled according to their genomic co-ordinates. For example, the Kozac sequence is found at positions 1809-1812 as described above. However, the Kozac sequence will not be located at these positions within the sequence fragment itself. The Anchor motif and Anchor position parameters allow the tool to orientate the input sequence fragment in the context of the entire genome. The anchor motif should be a conserved region, which occurs before (that is, "to the left of") of the first locus of interest. In BCP/PC sequences, the motif "AGATTA" at genomic co-ordinate 1750 is highly-conserved. The tool searches for the entire anchor motif in the first sequence in the input file. If the motif is not found in the first sequence, the tool reports an error. If the motif is found in the first sequence, the tool considers the first reside of this motif to have the genomic position indicated by the anchor position.

The loci of interest are entered into the Loci field of the input page. Individual loci may be separated by a single comma, with ranges separated by a dash. No other characters or spaces should be entered. For example, the ten loci of interest within the BCP/PC region would be specified as 1762,1764,1809-1812,1858,1862,18881,1896. This is achieved by entering values in to the Output grouping field on the input page. This field requires a list of numbers, separated by commas. No other characters or spaces should be entered. The numbers represent the number of loci to be grouped. In the example, 1762 and 1764 are considered as a unit, as are 1809-1812. By default, all sequences in the input file are processed. If only a subset of sequences should be processed, a regular expression can be entered into the Include/Exclude sequence ID pattern field. For example, if sequences from symptomatic patients are given FASTA IDs in the form >Patient001SYM and those of asymptomatic patients the form >Patient002NON, then either group may be included or excluded easily by specifing the appropriate regular expression.

Output Parameters

Several output parameters may be specified, as follows:

  • FASTA sequence IDs on the output page are trimmed to the length specified by the FASTA ID truncation length field.
  • If the check-box next to Sort sequences by ID is checked, sequences will be sorted alphabetically by the sequence ID.
  • Numerical values of residue distributions in the output tables will be shown as percentages if the check-box next to Output percentages instead of counts is checked; otherwise, raw count data will be shown.
  • To aid readability, values of zero in output tables can be omitted by checking the check-box next to Suppress zeros in ouput. If table data is copied from the output page and pasted into a spreadsheet, it may be necessary to include the zero values.

Full details of the "Mutation Distribution Graph Parameters" section are provided in the separate graph tutorial here (this link will open in a new browser window).

When the tool is used in Discovery Mode, it examines all the loci specified, but only selects for inclusion those loci at which variation occurs. This can be used to discover positions of interest by excluding positions at which no variation occurs. A single variant residue is sufficient for a locus to be included. When the Discovery Mode check-box is checked, input fields for several parameters, which are incompatible with this mode, will be removed from the input page. These fields will reappear when the check-box is unchecked.

Output Page

The output page consists of several sections, as follows.

Loci Distribution

This table displays the residue at each of the loci specified for each input sequence. In the example above, only the first 6 of 49 sequences are shown. If an output grouping was specified, the residues are separated into columns. In discovery mode, only variant residues are shown. Output grouping is not available in discovery mode.

Residue Distribution Summary

This table shows the distribution of residues at each locus. Values are either percentages (shown above) or raw counts.

Mutation Distribution Graph

If a reference motif was specified on the input page, a mutation distribution graph will be included on the output page. A detailed discussion of the mutation distribution graph is provided in the separate graph tutorial here (this link will open in a new browser window). Brielfy, the graph is a plot of the distribution of residues (as shown in the "Residue Distribution Summary" table above) after excluding the reference residue specified for each locus. The graph, therefore, shows non-reference residues at each locus and can be used to visualize the extent of variation found at each locus.

The plot above was produced by first running the tool with the example parameters discussed previously. No reference motif was specified, so no graph was produced. The motif distribution table (see below) was examined, and the motif which occurs most frequently was noted. The tool was then run a second time, with the noted motif AGTCATCGGG specified as the reference motif in the "Mutation Distriubtion Graph Parameters" section. The graph shows the distribution of residues for all sequences which do not match the reference motif specified. For example, at position 1812, the reference sequence is a T, as shown on the X-axis label below the bar. The blue bar indicates that, apart from the reference residue, only a resiude of C was found at this position, and that it occurred in approximately 22% of the sequences. At position 1810, apart from the reference resiude, both an A and a C occur, each in approximately 2% of the sequences. The raw data used to generate the plot can be downloaded as a CSV (comma-separated value) file by clicking on the link below the graph. CSV files can easily be imported into spreadsheet applications.

Motif Distribution

All specified loci are concatenated (joined) to produce a motif for each sequence. This motif is not a linear sequence of adjacent residues as would be found in the sequence data. Rather, it is the motif (pattern) produced by placing the residue found at each locus, for each sequence, next to each other to generate a motif. The "Motif Distribution" table shows the distribution of these motifs, sorted in descending order. For example, in the table above, the motif AGTCATCGGG occurs in 44.90% of sequences. This means that in 44.90% of the sequences, a residue of A was found at position 1762, a residue of G at 1764, and so on. A unique label (for example, L0001) is attached to each motif. The label for each motif is used on the X-axis of the graph. This has been done to avoid text overlapping, which may occur if the motifs are long.
