HVDRP Mutation Reporter Tool

Frequently Asked Questions

Is this tool free of charge?

Yes, this tool may be used by anyone, free of charge.

The site is slow or inaccessible. Is a mirror available?

The tool is hosted by the University of the Witwatersrand and under normal circumstances, should always be available and responsive. Please contact us if you continue to have difficulties in this regard. As this tool is under active development, no mirrors are available.

Can I download the tool to use offline?

As the tool is one component of a larger, integrated project, and follows the client-server model, it is not available for download as a standalone application. If you have a specific need in this regard, please contact us.

What data has been used to test and develop the tool?

The tool has been tested extensively with nucleotide sequence data from the hepatitis B virus, but data from any source can be processed.

Is a paper available? How should I cite the tool?

If you have used data or output generated by the tool in your publications or presentations, please cite the following paper:

Bell, T. G. and Kramvis, A. (2013) Mutation Reporter Tool: an online tool to interrogate loci of interest, with its utility demonstrated using hepatitis B virus. Virology Journal 10: 62.

Available online.

What input file formats are supported?

Input files must be plain text in FASTA format, and may contain either aligned nucleotide or aligned amino acid sequence data, from one or more samples.

I have sequence data in program X. How can I convert it to FASTA format?

Most bioinformatics programs should be able to export sequence data in FASTA format. In the exceptional situation that this is not the case, it may be possible to export the data into another program, which can export FASTA files.

Why must input files be in FASTA format? Why can't I upload files in format X?

FASTA is a universal bioinformatics file format. To ensure maximum compatability, we have selected this standard format for input files. More information on the FASTA file format is available at Wikipedia.

What are the anchor motif and anchor position? Why are they required?

The tool searches for the first occurrence of the anchor motif in the first sequence in the input file. The position at which the anchor motif starts is given the co-ordinate of the anchor position. This co-ordinate mapping allows the tool to find the positions of the specified loci within the sequence fragment data. The loci specified would typically be numbered according to their co-ordinates in the genome, whereas the sequence data would typically be a subgenomic fragment.

I don't understand the output grouping.

If no output grouping is specified, all the loci are displayed in a single column. The loci of interest can be grouped into logical units by specifying output grouping columns. For example, consider the following ten loci of interest: 1762, 1764, 1809-1812, 1858, 1862, 1888, 1896. The first two loci are interpreted as a unit, as are the next four loci. An output grouping of "2,4,1,1,1,1", for example, would place the data from the first 2 loci into the first column, followed by the data from the next 4 loci into the second column, etc. This allows the data to be analyzed and interpreted more easily.

What is Discovery Mode?

When the tool is used in "Discovery Mode", it examines all the loci specified, but only selects for inclusion those loci at which variation occurs. This can be used to discover positions of interest by excluding positions at which no variation occurs. A single variant residue is sufficient for a locus to be included. When the "Discovery Mode" check-box is checked, input fields for several parameters, which are incompatible with this mode, will be removed from the input page. These fields will reappear when the check-box is unchecked.

What is the motif distribution table and graph on the output page?

For this section of output, all of the specified loci are concatenated into a single motif. The distribution of this motif in the input data is tabulated and displayed as a graph.
