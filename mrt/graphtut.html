Mutation Reporter Tool


Graph Tutorial The tool will produce a mutation distribution graph only if a reference motif is specified on the input page. Graph output is disabled when Discovery Mode is selected. The graph plots the variation shown in the Residue Distribution Summary table. That is, loci are plotted on the X-axis and percentage frequency for residues at each locus are plotted on the Y-axis. Various options on the input page (see below for more details) allow for the graph output to be modified. The raw data used to create the graph is available for download as a CSV (comma-separated values) file by clicking the link below the table. A CSV file can easily be imported into a spreadsheet or statistical program.

Reference MotifIf a reference motif is specified, the tool will produce a graph. The reference motif is a string of the wild-type (reference) residues expected at each locus, in order. As such, a reference residue must be specified for each locus. The tool excludes (masks) the reference residue from the graph. This results in the graph showing the distribution of mutations (non-reference or non-wild-type residues) for each locus. It is not required that the reference be the wild-type however. Any motif can be entered as a reference, allowing for greater customization of the graph. As an example, if the input data is curated nucleotide sequence data which does not contain the residue "N", then specifying a reference of  N  for each locus will result in the tool plotting all residues for each locus, as none of the residues A, C, G or T will match the reference residue of "N" and will therefore be considered a mutation. The same result can be achieved for amino acid sequence data by specifying a reference of  -  or  ?  for each locus, with the proviso that the chosen character should not be present in the input sequence data.
Graph TitleText entered here will appear at the top of the graph.
Graph WidthThe width of the graph in pixels. Text on the graph scales with graph width. However, the text may be too large on very wide graphs showing a large number of loci. Likewise, the text may be too small on very narrow graphs. In these situations, it is recommended that the raw graph data be downloaded to plot with a program locally.
Graph HeightThe height of the graph in pixels.
Stacked BarsWhen this control is selected, the second and subsequent residues at a locus will be plotted vertically, producing a stacked bar. When this is unselected, multiple residues at a locus are plotted next to each other. If graphs from several different data-sets are to be compared downstream, it is recommended that this control be selected, so that the graphs produced are more easily compared, as the width of each locus will then be constant.
Suppress Conserved LociWhen this control is selected, loci at which only the reference residue is found will not be included in the graph. When this control is unselected, all specified loci will be plotted. Loci at which only the reference residue is found will be shown on the graph as an "empty" column without data. If graphs from several different data-sets are to be compared downstream, it is recommended that this control be unselected, so that the graphs produced are more easily compared, as each graph will then contain all specified loci.
Y-Axis Scaled to 100%When this control is not selected, the scale (range) of the Y-axis will be determined by the input data. For example, of the highest percentage frequency in the data is 50%, the Y-axis will scale to show a range between 0% and 50%. When this control is selected, the Y-axis will range from 0% to 100%, regardless of the range of the input data. If graphs from several different data-sets are to be compared downstream, it is recommended that this control be selected, so that the Y-axis of all the graphs are the same (100%). This avoids visual scaling confusion between the graphs.
